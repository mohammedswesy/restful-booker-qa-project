# 🧪 Restful Booker API Testing Project (QA)

## 📌 Overview

This project is a comprehensive **API testing project** for the Restful-Booker system, developed as part of a **Scrum-based QA simulation**.

It simulates a real-world QA workflow including:

* Requirement analysis
* Test case design
* API testing execution
* Validation & reporting

---

## 🎯 Objectives

* Validate API functionality and behavior
* Design **positive & negative test cases**
* Verify status codes and response structure
* Ensure proper error handling
* Apply real Scrum workflow in QA process

---

## ⚙️ Tools & Technologies

* 🧰 Postman – API testing & automation
* 🗂️ Jira – Test case management
* 📊 Microsoft Excel – Test design
* 📽️ PowerPoint – Presentation
* 🐙 GitHub – Version control

---

## 🧩 Scope of Testing

The following APIs were tested:

* 🔐 Authentication (`POST /auth`)
* ➕ Create Booking (`POST /booking`)
* 📄 Get Booking (`GET /booking`)
* 🔄 Update Booking (`PUT /booking/{id}`)
* ✏️ Partial Update (`PATCH /booking/{id}`)
* ❌ Delete Booking (`DELETE /booking/{id}`)

---

## 🧪 Testing Approach

* Positive Testing
* Negative Testing
* API Validation
* Status Code Verification
* Error Handling Testing
* Edge Case Testing

---

## 🚀 How to Run the Project

1. Import the Postman collection from `/postman`
2. Import the environment file
3. Set environment variables (`base_url`, `token`, `booking_id`)
4. Run requests manually or using Collection Runner

## 🏁 Results & Achievements

* ✅ Successfully tested all major booking APIs
* ✅ Covered positive & negative scenarios
* ✅ Validated responses and status codes
* ⚠️ Identified inconsistencies between API behavior and documentation

---
## 💡 Key Insights

- Some endpoints returned unexpected status codes compared to documentation  
- API behavior was inconsistent in certain edge cases  
- Importance of validating both response body and status codes  
- Real-world APIs may not always match their documentation
